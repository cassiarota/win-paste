# FineClipboard 设置页与快捷键修复设计

## 目标

- `Ctrl+Shift+B`（macOS 为对应自定义快捷键）只粘贴最近一条纯文本，不再连带打开历史弹窗。
- 设置窗口改为固定 Tab 导航，不再用纵向滚动串联分组。
- 截图快捷键与另外两个快捷键使用完全一致的可点击、录制和对齐方式。
- 密码设置只保留首次“设置密码”入口；设置后仅显示已设置状态，不提供修改或锁定入口。
- 不再提供“维护”分组；历史清空从列表右键菜单迁到托盘/菜单栏图标右键菜单。

## 设计

Windows 和 macOS 保持同样的六个设置 Tab：通用、历史与隐私、外观、快捷键、同步、密码。历史列表本身继续承载项目删除、收藏和列表归类操作，设置页不重复这些能力。

纯文本快捷键触发后，应用先写入纯文本剪贴板，再等待用户释放触发快捷键使用的修饰键，最后才注入系统粘贴键。这样不会把仍按住的 Shift 与注入的 `Ctrl+V` / `Cmd+V` 组合成历史弹窗快捷键。

托盘清空动作保留收藏项并要求二次确认；清空后立即刷新已经创建的历史弹窗模型。

## 验证

- 静态检查快捷键 ID、默认组合与处理函数的映射。
- 检查设置页不再包含外层 `ScrollViewer` / `ScrollView`，且六个分组均为 Tab。
- 检查历史列表右键菜单不再包含清空历史，托盘菜单包含该动作。
- 运行可用的 Windows/macOS 构建或语法检查，并运行 `git diff --check`。

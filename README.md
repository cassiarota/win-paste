# FineClipboard

一个 **Windows 与 macOS** 上的**剪贴板历史管理器**。

系统自带的剪贴板(Windows 的 `Win+V`、macOS 没有持久历史)**一重启就清空**,用起来也不顺手。FineClipboard 把你
复制过的**文本 / 图片 / 文件**都本地持久化保存下来,随时用一个全局快捷键调出、搜索、选中、粘贴,并且支持
**加密的密码管理**。

> 平台:Windows 10 / 11(64 位)与 macOS 13 (Ventura) 及以上。程序常驻系统托盘 / 菜单栏。
> Windows 版自带运行时,**无需安装 .NET**;macOS 版 Apple Silicon / Intel 通用。

## 下载与安装

到 [**Releases**](https://github.com/cassiarota/fine-clipboard/releases) 或 **官网**(Cassian Engines 门户首页点
**FineClipboard** 卡片进入)下载。

### Windows

| 文件 | 适合 | 用法 |
|---|---|---|
| `FineClipboard-*-Setup.exe` | 大多数人 | 双击安装(**无需管理员、无 UAC**,装到当前用户目录),可勾选开机自启 |
| `FineClipboard-*-win-x64.zip` | 想免安装 / 放 U 盘 | 解压后双击 `FineClipboard.exe` 直接运行 |

启动后**没有主窗口**——程序在系统托盘(屏幕右下角)。右键托盘图标可打开菜单。

### macOS

下载 `FineClipboard-*-mac.zip`,解压得到 `FineClipboard.app`,拖到「应用程序」即可。

> 该应用未经 Apple 公证,**首次打开**可能提示“无法验证开发者”。解决:**右键(或 Control 点按)图标 →「打开」**,
> 在弹窗里再点一次「打开」即可;之后正常双击。或在终端执行
> `xattr -dr com.apple.quarantine /Applications/FineClipboard.app`。
>
> 首次使用还需在 **系统设置 → 隐私与安全性 → 辅助功能** 里勾选 FineClipboard,程序才能模拟 `Cmd+V` 自动粘贴。

启动后程序在**菜单栏**(屏幕顶部栏的剪贴板图标),点击图标打开菜单。

## 快速上手

| 快捷键(默认) | Windows | macOS | 作用 |
|---|---|---|---|
| 打开历史 | **Ctrl + Shift + V** | **⌘ + Shift + V** | 弹出剪贴板历史框,搜索并选择要粘贴的内容 |
| 纯文本粘贴最近一条 | **Ctrl + Shift + B** | **⌘ + Shift + B** | 把**最近一条**历史以**纯文本**(去格式)粘贴到当前窗口 |

在历史弹窗里:

- **按 `1`~`9`** 直接粘贴对应序号的条目(前 9 条会显示数字角标;开始输入搜索后该功能让位给输入)
- `↑` / `↓` 选择,`Enter` 粘贴,`Cmd/Ctrl + Enter` 或 `Shift + Enter` 以纯文本粘贴,`Esc` 关闭
- 顶部输入框可**实时搜索**;上方 `全部 / 文本 / 图片 / 文件 / 置顶 / 片段 / 密码` 以及你的**自定义列表**标签可**筛选**
- **右键**任意一条:粘贴 / 纯文本粘贴 / 编辑后粘贴 / 转换后粘贴 / 复制 / 打开链接 / 定位文件 / 图片另存 / 置顶 / 移到列表 / 删除(按内容类型显示)
- 直接把一条**拖拽**到其它应用即可(文本 / 图片 / 文件)
- 弹窗会出现在**鼠标所在的显示器**上;点击别处自动关闭

> 说明:Windows 的 `Win+V` 已被系统占用,所以默认沿用 `Ctrl+Shift+V` / `Ctrl+Shift+B`;macOS 对应 `⌘` 组合。
> 两个快捷键都**可以自己改**(见下)。

## 功能一览

Windows 与 macOS **功能一致**:

- 自动保存**文本 / 图片 / 文件**三类内容,**跨重启持久化**,相同内容自动去重并置顶
- **搜索** + **类型分组**,以及 `1`~`9` **数字键快速粘贴**;**图片 OCR**:截图 / 图片里的文字会被本地识别(Windows OCR / macOS Vision,离线),可直接被搜索命中
- **格式保真**:复制带格式的文本会同时保存 HTML / RTF,默认带格式粘贴,`Ctrl/⌘ + Enter` 仍以纯文本粘贴
- **截图**:托盘 / 菜单栏「截图」可截取**区域 / 窗口(磁吸) / 全屏**(默认 `Ctrl/⌘ + Shift + A`),截图默认进剪贴板并入历史(自动 OCR),右键即可另存为文件并选择目录
- 粘贴过的条目自动排到最前(最近使用优先);可**置顶**常用条目
- **常用片段**:把签名、地址、模板等存成命名片段,在「片段」标签随时粘贴,永不过期
- **自定义列表**:新建列表,右键历史条目「移到列表」分组管理(列表里的条目不会被自动清理)
- **加密密码管理**:专门的「密码」标签,主密码 + AES-256-GCM 加密保存;查看 / 粘贴需输入主密码,密码**绝不进历史**(见下)
- **暂停记录(隐私模式)**:在「设置 → 历史与隐私」中一键暂停,处理敏感内容时不留痕
- **拖拽**:从弹窗把条目直接拖到其它应用(文本 / 图片 / 文件)
- **内容动作**:右键可直接打开链接、在资源管理器 / 访达定位文件、把图片另存为
- **粘贴前编辑 / 转换**:贴出前临时修改文字,或一键去空格 / 合并单行 / 大小写
- **复制(放回剪贴板)**:只把条目放回剪贴板而不立即粘贴
- **自定义快捷键** + 可选**提示音**
- **深色 / 浅色 / 跟随系统**主题,弹窗大小可调
- **多显示器 + 高 DPI**:弹窗跟随光标所在屏幕,缩放自适应
- **过期自动清理** + 可配置**历史上限**(200 / 500 / 1000 / 5000 条)
- **排除规则**:指定某些程序(如密码管理器),它们的复制不被记录
- **云同步(跨设备 · VIP)**:在 Windows 与 macOS 之间端到端加密同步历史(见下)
- **自动检查更新**:启动时检查新版本,有更新时提示,一键打开下载

## 云同步(跨设备 · 需 VIP)

在多台设备(Windows / macOS)之间同步剪贴板历史。在「**设置 → 同步 → 打开云同步设置…**」进入设置。

1. **注册 / 登录**一个账号(邮箱 + 密码)。
2. **开通 VIP**:输入**激活码**兑换(目前 VIP 仅通过激活码开通)。
3. 设置一个**同步口令**:所有设备用**相同的邮箱与同步口令**,内容会用它派生的密钥**端到端加密**后再上传。
4. 勾选「开启自动同步」即可。客户端只上传**有变化**的条目(增量同步),并按游标拉取其他设备的更新;冲突按「后写覆盖」处理。

> **隐私与边界(务必知悉):**
> - **端到端加密**:同步口令**不会上传**,服务器**只保存密文**,无法读取你的剪贴板内容;忘记同步口令将无法解密云端数据。
> - **跨端同步强制不包含密码数据**——加密密码库的任何内容**绝不参与同步**,只在本机保存。
> - 同步**只保留文本与图片**,不同步大文件;云端数据**最长保留半年**(若你设置的历史过期时间更短则以更短为准)。

## 设置

右键托盘图标(Windows)/ 点击菜单栏图标(macOS)→「设置…」。暂停记录、云同步与锁定密码都集中在设置中。

- **开机自启动** — 登录时自动启动
- **复制时播放提示音**
- **暂停记录(隐私模式)** — 临时停止写入剪贴板历史
- **最多保留** — 非置顶历史的条数上限(200 / 500 / 1000 / 5000)
- **管理常用片段** — 新建 / 编辑 / 删除可复用的命名文本片段
- **管理列表** — 新建 / 重命名 / 删除自定义列表
- **密码保护** — 设置 / 修改主密码,管理加密保存的密码(见下)
- **锁定密码** — 立即锁定密码库,下次查看 / 粘贴需重新输入主密码
- **历史过期时间** — 1 / 7 / 30 / 90 天或永不;超时的非置顶记录会被自动清理
- **排除规则** — 每行一个程序名(Windows 进程名如 `KeePass`;macOS 应用名如 `1Password`),来自这些程序的复制不被记录
- **外观** — 主题(跟随系统 / 浅色 / 深色)与弹窗大小(小 / 中 / 大)
- **自定义快捷键** — 见下
- **清空历史** — 可选保留置顶项,或全部清空

### 密码管理

把密码加密保存在本地,需要时凭主密码取用。

1. 在「设置 → 密码保护」点 **设置主密码**(首次)。主密码用于加密,**不会被保存**,忘记将无法找回已存密码。
2. 点 **管理密码…**(需输入主密码解锁),在里面新建 / 编辑 / 删除条目(名称 + 密码)。
3. 平时在历史弹窗的 **「密码」标签** 里能看到条目名称(密码显示为 `••••••`)。
4. 选中某条按 `Enter` 或双击粘贴时,会要求输入主密码;解锁后即可粘贴,**密码不会写入剪贴板历史**。
5. 在「设置 → 密码」里的 **锁定密码** 可立即上锁;重启程序后也需重新解锁。

> 安全实现:主密码经 **Argon2id**(内存困难型 KDF,64 MiB / 3 轮 / 4 通道)派生密钥,密码用 **AES-256-GCM**
> 加密后存入本地数据库;本地只保存随机盐、KDF 标记和一个校验令牌,主密码本身从不落盘。

### 自定义快捷键

1. 打开「设置」,找到**快捷键**一栏。
2. 点击对应按钮,它会变成「按下快捷键…」。
3. 直接按下你想要的组合键(**需要包含至少一个修饰键**:Windows 为 `Ctrl/Shift/Alt/Win`,macOS 为 `⌘/⌥/⌃/⇧`)。
4. 松开即自动保存并立即生效;按 `Esc` 可取消。
5. 如果该组合键已被占用或与另一个快捷键冲突,会提示你换一个,原快捷键保持不变。

## 数据位置与卸载

- **Windows** 历史数据库:`%LOCALAPPDATA%\FineClipboard\`(SQLite)。
- **macOS** 历史数据库:`~/Library/Application Support/FineClipboard/`(SQLite)。
- 删除该目录即清空全部历史与设置。
- 卸载:Windows 安装版从「设置 → 应用」卸载,便携版删文件夹;macOS 把 `FineClipboard.app` 拖到废纸篓。
  卸载不会自动删数据目录,可手动删除上面的路径。

## 数据安全与加密

历史数据库里的**剪贴板内容(文本 / 图片)与密码都加密保存**,文件里读不到明文:

- **剪贴板历史**:用一把随机 256 位密钥做 **AES-256-GCM** 加密(预览也加密,去重哈希改为带密钥的 HMAC,
  无法被反推内容)。这把密钥由系统密钥库托管——**Windows 用 DPAPI**(绑定当前 Windows 账户)、
  **macOS 用钥匙串**(`AfterFirstUnlock`)——封存后保存在本地,**不需要你输入任何密码**,后台记录照常工作。
  数据文件被拷到别处 / 进了备份 / 被同机其它账户读取,都无法解密。
- **密码库**:在上面的基础上再加一层**主密码零知识加密**(Argon2id + AES-256-GCM,主密码从不落盘),
  连系统密钥库托管的那把密钥也无法解开。
- 数据目录 / 文件已限制为**仅当前用户可读**(Windows 位于用户私有的 `LocalAppData`;macOS 设为 `700/600`)。

> 说明:本地静态加密无法防御**以你自己账户身份运行的恶意软件**(它能直接读实时剪贴板,任何方案都防不住)。
> 它防的是文件被拷走 / 备份或云同步泄露 / 同机其它账户 / 硬盘丢失或被克隆等场景。
> 建议同时开启系统的**全盘加密**(macOS 的 FileVault、Windows 的 BitLocker)作为基础保护。

## 注意事项

- 粘贴注入在 Windows 上**无法作用于以管理员权限运行的窗口**;在 macOS 上需要授予**辅助功能**权限。
- 自定义的全局快捷键是**系统级**的,会覆盖个别程序内同样组合键的功能;如有冲突,换一个组合即可。
- 粘贴密码后,该密码会像正常粘贴一样**短暂留在系统剪贴板**里(直到你下次复制);它不会进入本程序的历史。
- **忘记主密码无法找回**已保存的密码(这是加密的应有之义),请务必牢记。

## 从源码构建(开发者)

### Windows(C# / .NET 8 / WPF)

需要 [.NET 8 SDK](https://dotnet.microsoft.com/download/dotnet/8.0)(Windows)。

```powershell
# 开发运行
dotnet run --project src/FineClipboard/FineClipboard.csproj

# 发布为自带运行时的单文件 exe
dotnet publish src/FineClipboard/FineClipboard.csproj -c Release -r win-x64 `
  --self-contained true -p:PublishSingleFile=true -p:IncludeNativeLibrariesForSelfExtract=true
```

使用 Win32(`AddClipboardFormatListener`、`RegisterHotKey`、`SendInput`)实现剪贴板监听、全局热键与粘贴注入,历史存于 SQLite。

### macOS(Swift / SwiftUI / AppKit)

需要 Xcode 命令行工具(`xcode-select --install`)。

```bash
cd mac
swift build -c release          # 或 ./build-app.sh 0.9.0 直接打包成 FineClipboard.app
```

使用 Carbon(`RegisterEventHotKey`)做全局热键、`NSPasteboard` 轮询监听剪贴板、`CGEvent` 模拟 `Cmd+V` 粘贴、
`NSStatusItem` 菜单栏图标。历史同样存于 SQLite 并加密(密钥存钥匙串),密码库用 Argon2id(内置 PHC 参考实现,
见 `mac/Sources/CArgon2/`)+ CryptoKit 的 AES-256-GCM 加密。
